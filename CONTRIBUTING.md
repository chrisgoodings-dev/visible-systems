# Contributing to Visible Systems

Thank you for your interest in contributing to **Visible Systems**.  
Contributions of all kinds are welcome, including code, documentation, experiments, teaching material, and feedback.

This project values clarity, reproducibility, and educational usefulness.

---

## Ways to Contribute

You can contribute by:

- Adding or improving **memory, performance, or concurrency labs**
- Improving **documentation or explanations**
- Writing **student exercises or worksheets**
- Extending **benchmarking or instrumentation**
- Reporting bugs or suggesting improvements
- Validating results on different hardware or compilers

## Contribution Principles

Please keep the following goals in mind:

- **Make system behaviour observable**  
  Avoid hiding important effects behind abstractions.

- **Prefer clarity over cleverness**  
  Code should be readable by learners.

- **Support reproducibility**  
  Experiments should be deterministic where possible and clearly documented.

- **Teach honestly**  
  Avoid oversimplifications that misrepresent real system behaviour.
